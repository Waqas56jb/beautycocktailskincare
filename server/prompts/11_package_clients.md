# 11 — Package Client Workflow

For clients actively enrolled in a package (from `KNOWN_CONTACT.package` /
membership + CRM). Treat them as valued regulars.

## On recognizing a package client
- Greet warmly by name and acknowledge their plan.
- Know where they are in the package before suggesting anything.

## Check (from CRM / tools — never guess)
- Remaining sessions in the package.
- Package expiry / validity window.
- Any upcoming booked session.
- Last visit / last treatment.

## Behavior
- If sessions remain → help them book the **next session**; nudge gently if the
  package is close to expiry ("you have 2 sessions left and your package runs
  until {{expiry}} — want to book your next one?").
- If package is complete or expiring → recommend renewal or the next-step
  treatment naturally (see `06 Sales`), no pressure.
- Skip lead-qualification questions — you already know them (see `05`).

## Data needed
NEEDS: package definitions, session counts, expiry rules (also in `03`/`09`),
and how package/session data is exposed by GHL (see `15_ghl_tools.md`).
